# SEBI Reference Extraction Agent

A CLI agent that extracts every reference to external documents from Securities and Exchange Board of India (SEBI) regulatory PDFs, producing graph-ready JSON with verbatim evidence.

## Architecture

```
┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐
│ 1 Extract│→ │ 2 Regex  │→ │ 3 Gemini │→ │ 4 Alias  │→ │ 5 Title  │→ │ 6 Verify │
│ PyMuPDF  │  │candidates│  │ per-page │  │resolve   │  │CSV lookup│  │+ dedupe  │
│+ zones   │  │+ hints   │  │JSON out  │  │("said X")│  │(optional)│  │          │
└──────────┘  └──────────┘  └──────────┘  └──────────┘  └──────────┘  └──────────┘
     │                                                                      │
     ▼                                                                      ▼
 pages[]                                                         mentions.json
                                                                 documents.json
                                                                 edges.json
```

The pipeline is intentionally **hybrid**. Regex seeds the LLM with likely candidates (improving recall on long PDFs, anchoring the model's attention to the right spans), but the LLM is the final judgment on whether a match is a genuine external reference and what its canonical form is. Every extraction is **verifiable** — the agent must copy `evidence_text` verbatim from the page, and stage 6 drops anything whose evidence cannot be located on its claimed page.

## Quick start

```bash
pip install -r requirements.txt
cp .env.example .env                            # then paste your key into .env
python -m src.run --pdf sample.pdf --out output/
```

Get a Gemini API key at https://aistudio.google.com/apikey. The `.env` file is gitignored — each developer keeps their own. You can also `export GEMINI_API_KEY=...` or pass `--api-key` instead of using `.env`.

Outputs are written to `output/mentions.json`, `output/documents.json`, `output/edges.json`.

Run `pytest tests/ -v` to exercise the unit tests. Tests do not require PyMuPDF, a Gemini API key, or network access — both heavy deps are imported lazily so the unit suite runs on any Python 3.11+ environment.

## Output schema

See [`CLAUDE.md`](CLAUDE.md) for the full field reference. Short version:

- **`mentions.json`** — one row per occurrence (a document cited three times → three rows). Carries `source_page`, `display_page`, `mention_text`, `evidence_text`, `doc_type`, `doc_id`, `date`, `title`, `relation_type`, `confidence`, `resolution_status`, `extraction_source`.
- **`documents.json`** — deduplicated canonical references with `mentioned_on_pages`, `times_mentioned`, `relation_types`.
- **`edges.json`** — flat graph export: `(source, relation, target, target_type, source_page, evidence)`.

## Eval methodology

`src/eval.py` computes:

| Metric             | What it measures                                                                     |
|--------------------|--------------------------------------------------------------------------------------|
| Document P/R/F1    | Set-match over canonical documents (keyed by `doc_id` or `(doc_type, normalized title)`) |
| Page accuracy      | Fraction of matched mentions whose `source_page` matches gold                        |
| Title accuracy     | Strict-equality and fuzzy (≥0.85 SequenceMatcher ratio) over matched documents       |
| Hallucination rate | Fraction of mentions whose `evidence_text` is not present on the claimed page        |
| Unresolved rate    | Fraction of mentions where `title` is null                                           |
| Sliced P/R/F1      | Per-`doc_type` breakdown (circular vs regulation vs act vs …)                        |

### Gold-label workflow

Gold annotations live in `tests/gold/*.json`. To avoid circular evaluation (writing gold with the same prompt the agent uses), gold drafts are generated by a **different prompting strategy**:

1. `scripts/draft_gold.py` sends the **whole document in a single pass** to Gemini with a stricter system prompt that emphasizes cross-page reasoning.
2. `scripts/compare_and_merge.py` diffs agent output against the draft, auto-accepts agreements, and surfaces only disagreements for a human reviewer.

Agreement between two independent prompting strategies is a stronger signal than agreement with the agent's own output, so this keeps the eval honest.

```bash
python scripts/draft_gold.py --pdf doc.pdf --save tests/gold/doc.json
python -m src.run --pdf doc.pdf --out output/
python scripts/compare_and_merge.py \
  --agent-output output/documents.json \
  --draft-gold tests/gold/doc.json \
  --save disagreements.json
python -m src.eval --gold tests/gold --predictions output/
```

## Limitations

- **OCR not supported.** Scanned PDFs without a text layer yield empty pages. v2 will add a Tesseract fallback.
- **Zone detection is heuristic.** Footnote / table detection uses whitespace patterns, not visual layout. Expect some leakage between body and footnote zones in dense typography.
- **Alias resolution is local.** "The said Regulations" is resolved against the last mention of type `regulation` — it cannot follow long-distance coreference across annexures or cross-document boundaries.
- **Title normalization is minimal.** `SEBI (LODR) Regulations, 2015` and `SEBI Listing Obligations and Disclosure Requirements Regulations, 2015` may deduplicate separately unless both surface forms appear on the same run.
- **No SEBI metadata ingestion yet.** `scripts/scrape_sebi_index.py` is a stub. `src/resolve.TitleResolver` accepts any CSV you supply with columns `doc_id,title,date,doc_type`.
- **Self-reference filter is English-only** and keyed on a short closed list.
- **Per-page LLM extraction misses cross-page context.** A reference defined on page 3 and cited by shorthand on page 40 resolves only if the first-mention title was captured earlier. The draft-gold full-document pass is the intended cross-check.
- **No retry / backoff on Gemini errors.** A single failed page is logged and treated as empty; rerun the pipeline to re-attempt.

## v2 roadmap

- OCR fallback (Tesseract) for scanned PDFs
- Real `scrape_sebi_index.py` populating canonical IDs/titles from sebi.gov.in
- Visual zone detection using PyMuPDF block coordinates instead of whitespace heuristics
- Title normalizer (strip "SEBI", expand abbreviations, unify punctuation) before dedup
- Cross-page co-reference memory for long annexures
- Confidence calibration from held-out gold
- Optional embedding-based dedup for `documents.json`
- Retry / exponential backoff on Gemini API errors
